# Monitor Workspace

The centralized Observability and Telemetry hub for the `tupynambalucas.dev` monorepo.

This workspace implements the **Edge Observability** pattern using the **OpenTelemetry Gateway architecture**. It provides a unified ingestion point for all metrics, logs, and distributed traces generated by our applications and AI agents.

## Architecture

- **Gateway (`/gateway`)**: OpenTelemetry Collector acting as the single front-door for telemetry ingestion (OTLP).
- **Services (`/services`)**:
  - **Prometheus**: Time-series database for metrics.
  - **Grafana**: Visualization and dashboards.
  - **Tempo/Jaeger** (Optional): Distributed tracing backends.

## Network Topology

This workspace creates an external Docker bridge network called `tupynambalucas-monitor-net`.
Other workspaces (like the MCP workspace) connect to this network to securely push telemetry data to the OpenTelemetry Collector without exposing internal backend ports to the host machine.
