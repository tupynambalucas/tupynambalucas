The Profile workspace contains the Node.js and TypeScript-based automation engine that collects developer metrics from GitHub and compiles them into custom SVG cards, dynamically updating the repository's root README.md.

---

## Directory Structure

```text
profile/
├── dist/             # Compiled JavaScript output and generated SVGs
├── src/              # TypeScript source files implementing the statistics engine
│   ├── clients/      # API clients (GitHub and Git Clone Fallback)
│   ├── renderers/    # SVG stats visual card compilation
│   ├── templates/    # Embedded SVG and Markdown templates
│   ├── utils/        # Token substitution and glob matching utilities
│   ├── config.ts     # CLI configuration loading
│   ├── types.ts      # TypeScript interfaces and Zod schemas
│   └── index.ts      # Entry point coordinating card generation and README compiling
├── package.json      # Workspace dependencies and scripts
└── tsconfig.json     # Configuration for TypeScript compilation and type resolution
```

---

## Scoped Operations

Run these scripts from the monorepo root to manage the profile generator stack:

- `pnpm --filter @tupynambalucas/profile build`: Compiles TypeScript files into `dist/`.
- `pnpm --filter @tupynambalucas/profile dev`: Runs the compiler in watch mode.
- `pnpm --filter @tupynambalucas/profile typecheck`: Validates workspace config types.
- `pnpm --filter @tupynambalucas/profile start`: Runs the compiled statistics generator CLI.
