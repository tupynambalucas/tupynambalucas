O workspace Tools fornece utilitários de desenvolvimento isolados para controle de versão Git e automação do GitHub CLI. Ao empacotar ferramentas de linha de comando dentro de containers Docker padronizados, garante-se a paridade do ambiente para operações de repositório entre estações locais e pipelines de CI/CD.

Esta seção cobre:

- **Containers Git e GitHub CLI**: Ambientes conteinerizados padronizados para execução de tarefas de repositório locais e remotas.
- **Paridade de Ambiente**: Eliminação de bugs causados por diferença de versão de ferramentas como Git Flow e extensões da CLI do GitHub.
- **Automação de Fluxos de Trabalho**: Comandos específicos para iniciar, parar e resetar os containers de ferramentas via Docker Compose.
