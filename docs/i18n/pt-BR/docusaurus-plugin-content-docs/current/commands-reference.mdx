# Referência de Comandos

Este documento serve como a **Única Fonte da Verdade** (Single Source of Truth) para todo comando CLI, script PNPM, comando Docker e instrução de orquestração usados no monorepo Elo Orgânico.

## Monorepo Orchestration

Scripts usados para gerenciar dependências, limpeza e tarefas no nível do workspace.

- **`pnpm install`**: Instala todas as dependências no monorepo.
- **`pnpm install --frozen-lockfile`**: Instala dependências estritamente a partir do `pnpm-lock.yaml` (usado em CI/CD).
- **`pnpm project:clear`**: Limpa todos os `node_modules` e caches de build usando `npkill`.
- **`pnpm add <package> --catalog`**: Adiciona um pacote ao catálogo compartilhado do PNPM.
- **`pnpm add <package>@catalog: --filter <app-name>`**: Instala um pacote catalogado em um workspace específico.

## Validation & Quality

Scripts para garantir a qualidade do código e a tipagem estrita.

- **`pnpm typecheck`**: Executa a validação do TypeScript em paralelo em todos os workspaces via Turborepo.
- **`pnpm typecheck --filter=@elo-instance/*`**: Executa o typecheck apenas para o contexto delimitado (bounded context) Instance.
- **`pnpm lint`**: Executa a validação do ESLint em todos os workspaces.
- **`pnpm build`**: Compila um build de produção para todas as aplicações e pacotes.
- **`pnpm --filter @elo-instance/core build`**: Faz o build de um pacote específico.

## Instance (Community)

Comandos específicos para o contexto delimitado `@elo-instance/*`.

- **`pnpm instance:up`**: Inicia a infraestrutura local (MongoDB Replica Set + Redis) via `compose.dev.yaml`.
- **`pnpm instance:dev`**: Inicia a stack completa de desenvolvimento (Infraestrutura + API na porta 3000 + Web na porta 5173). Chama internamente `instance:up` primeiro.
- **`pnpm instance:down`**: Para a infraestrutura local e encerra forçadamente as portas 3000 e 5173.
- **`pnpm instance:reset`**: Destrói os volumes da infraestrutura local (limpando o banco de dados) e reinicia a stack.
- **`pnpm instance:staging`**: Compila e orquestra o ambiente de staging usando `compose.prod.yaml` e `.env.staging`.
- **`pnpm instance:staging:down`**: Para a topologia de staging.
- **`pnpm instance:prod`**: Compila e orquestra o ambiente de produção usando `compose.prod.yaml` e `.env.prod`.
- **`pnpm instance:prod:down`**: Para a topologia de produção.

## Portal (Platform)

Comandos específicos para o contexto delimitado `@elo-portal/*`.

- **`pnpm portal:up`**: Inicia a infraestrutura local para o portal.
- **`pnpm portal:dev`**: Inicia a stack completa de desenvolvimento do portal (API na porta 3001 + Web na porta 5174).
- **`pnpm portal:down`**: Para a infraestrutura local e encerra forçadamente as portas 3001 e 5174.
- **`pnpm portal:reset`**: Destrói os volumes da infraestrutura local do portal.
- **`pnpm portal:staging`**: Compila e orquestra o ambiente de staging do portal.
- **`pnpm portal:staging:down`**: Para a topologia de staging do portal.
- **`pnpm portal:prod`**: Compila e orquestra o ambiente de produção do portal.
- **`pnpm portal:prod:down`**: Para a topologia de produção do portal.

## Documentation

Comandos para a Base de Conhecimento em Docusaurus (`docs/`).

- **`pnpm docs:dev`**: Inicia o servidor local de desenvolvimento do Docusaurus com HMR.
- **`pnpm docs:build`**: Compila a documentação em HTML/CSS estático para produção.

## Studio & Tools

Comandos para design e agentes de IA.

- **`pnpm penpot:up`** / **`pnpm penpot:down`**: Inicia ou para o Penpot Studio self-hosted.
- **`pnpm penpot:aide:up`** / **`pnpm penpot:aide:down`**: Inicia ou para o Penpot Aide AI Bridge.

## Git & Version Control

Fluxos de trabalho para branching, commits e releases.

- **`git flow feature start <name>`**: Inicia uma nova branch de feature a partir da `develop`.
- **`git push -u origin feature/<name>`**: Faz o push de uma branch de feature para o repositório remoto.
- **`gh pr create --base develop --fill --web`**: Cria um Pull Request apontando para a branch `develop`.
- **`gh pr merge --squash --delete-branch`**: Mescla um PR usando a estratégia squash-and-merge.
- **`git flow hotfix start <name>`** / **`git flow hotfix finish <name>`**: Gerencia correções de emergência diretamente na branch `main`.
- **`pnpm version:changeset`**: Gera um arquivo de changeset para rastrear incrementos de versão e changelogs.

## Docker & Infrastructure

Comandos diretos da CLI do Docker para depuração e manutenção.

- **`docker ps --filter "name=elo-instance"`**: Lista os containers em execução para o contexto instance.
- **`docker compose -f instance/compose.prod.yaml --env-file instance/.env.prod logs -f`**: Exibe os logs da stack de produção em tempo real.
- **`docker logs elo-instance-api-prod`**: Inspeciona os logs de um container específico.
- **`turbo prune`**: Executa o prune do workspace do monorepo para builds minimalistas no Docker.
- **`pnpm deploy`**: Usado dentro dos Dockerfiles para extrair apenas dependências compiladas de um pacote específico.
