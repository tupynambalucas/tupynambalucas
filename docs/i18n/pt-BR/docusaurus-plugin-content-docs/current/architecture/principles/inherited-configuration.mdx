# Configuração Herdada

A Configuração Herdada centraliza a governança de ferramentas e regras de compilação na raiz do monorepo, permitindo que as aplicações downstream estendam configurações compartilhadas sem duplicar definições.

---

## 1. Conceito Arquitetural & Visão Geral

Gerenciar o linting, a compilação e a formatação individualmente por pacote em um monorepo inevitavelmente leva a divergências nas configurações das ferramentas e estilos de código inconsistentes. Centralizar as configurações das ferramentas na raiz do repositório e estendê-las em cada workspace individual garante diretrizes estritas e unificadas de qualidade de código, reduzindo drasticamente o boilerplate de configuração de cada workspace.

Esta estrutura permite impor padrões rigorosos no nível do monorepo (como formatação de código consistente, checagens estritas do TypeScript e limites de segurança de pacotes) através de um arquivo de configuração central único, mantendo todas as aplicações e pacotes limpos e previsíveis.

---

## 2. Utilização Específica no Projeto

No Elo Orgânico, todas as ferramentas e configurações são centralmente orquestradas a partir da raiz:

### A. Governança Unificada do ESLint

As regras do ESLint 9 (Flat Config) são centralizadas na raiz do monorepo. Os pacotes individuais importam e estendem essa configuração base em seus arquivos locais `eslint.config.js`, garantindo que limites de segurança, guias de estilo e padrões de qualidade de código sejam aplicados uniformemente em todos os projetos:

```javascript
// eslint.config.js local estendendo a Flat Config da raiz
import baseConfig from '../../eslint.config.js';

export default [
  ...baseConfig,
  {
    files: ['**/*.ts'],
    rules: {
      // Sobrescritas específicas do pacote local se necessário
    },
  },
];
```

### B. Configurações TypeScript em Cascata

Os arquivos `tsconfig.json` de nível raiz definem o nível de rigor base do compilador:

- `strict: true`
- `noImplicitAny: true`
- `noFallthroughCasesInSwitch: true`

Os workspaces estendem essas configurações compartilhadas através da propriedade `extends`, customizando apenas bibliotecas de tempo de execução específicas ou ambientes de destino (como contextos Node.js vs. DOM/navegador):

```json
// instance/apps/web/tsconfig.json
{
  "extends": "../../../tsconfig.base.json",
  "compilerOptions": {
    "target": "ES2022",
    "lib": ["DOM", "DOM.Iterable", "ES2022"]
  }
}
```

### C. Regras de Formatação Consolidadas

O estilo do código é governado globalmente por um único arquivo de configuração do Prettier na raiz do monorepo. As regras do Prettier são aplicadas consistentemente a todos os arquivos TypeScript, JSON, CSS e Markdown em toda a estrutura do workspace, garantindo um estilo visual completamente unificado sem regras ad-hoc.

### D. Governança de Orquestração de Tarefas

O Turborepo utiliza o arquivo `turbo.json` na raiz para modelar o grafo de dependências do workspace. Isso unifica as definições de execução de tarefas (como `build`, `lint`, `typecheck`) e garante que as otimizações de cache funcionem de maneira idêntica em todos os workspaces.
