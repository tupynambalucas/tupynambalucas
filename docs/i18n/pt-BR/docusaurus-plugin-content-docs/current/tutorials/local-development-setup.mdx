# Configuração de Desenvolvimento Local

Neste tutorial, nós configuraremos o monorepo tupynambalucas.dev em sua máquina local. Ao final desta lição, você terá o ambiente de desenvolvimento completo rodando e pronto para você construir novos recursos.

## Pré-requisitos

Antes de começarmos, certifique-se de ter os seguintes itens instalados:

- Node.js versão 22
- PNPM versão 11
- Docker e Docker Compose
- Git

## Passo 1: Clonar o Repositório

Primeiro, nós clonaremos o repositório em nosso espaço de trabalho local e navegaremos até ele.

```bash
git clone https://github.com/tupynambalucas/tupynambalucas.dev.git
cd tupynambalucas.dev
```

Você deverá ver uma estrutura de diretórios contendo pastas como `docs`, `hub`, `cortex`, `profile` e `tools`.

## Passo 2: Instalar Dependências

Em seguida, precisamos instalar as dependências do projeto. Como este é um monorepo, usaremos o PNPM para instalar tudo de uma vez.

Execute o seguinte comando:

```bash
pnpm install
```

Quando terminar, você verá uma saída semelhante a esta:

```
Packages: +1245
++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++
Packages are hard linked from the content-addressable store to the virtual store.
...
Done in 12.4s
```

## Passo 3: Configurar Variáveis de Ambiente

Agora, precisamos configurar nossa variável de ambiente local. Copiaremos o arquivo `.env.example` de amostra para criar nossas variáveis de ambiente ativas.

```bash
cp .env.example .env
```

## Passo 4: Iniciar a Infraestrutura Local

Com nossas dependências instaladas e ambiente configurado, inicializaremos os serviços de backend necessários (como o banco de dados) usando o Docker Compose.

```bash
pnpm infra:up
```

Aguarde alguns instantes. Você deverá ver o Docker baixando as imagens e iniciando os containers.

```
[+] Running 2/2
 ✔ Container mongodb  Started
 ✔ Container db-init  Started
```

## Passo 5: Iniciar o Servidor de Desenvolvimento

Finalmente, estamos prontos para iniciar a aplicação. Usaremos o comando do Turborepo para executar os servidores de desenvolvimento em todos os workspaces.

```bash
pnpm dev
```

Você verá a saída indicando que vários serviços estão iniciando. Procure por linhas como:

```
@tupynambalucas-hub/web: dev:   ➜  Local:   http://localhost:3000/
@tupynambalucas/docs: dev:      [SUCCESS] Docusaurus website is running at: http://localhost:3001/
```

Abra seu navegador e navegue até `http://localhost:3000`. Você deverá ver a interface do Hub carregada com sucesso.

## Conclusão

Parabéns! Nós clonamos o repositório com sucesso, instalamos as dependências, iniciamos a infraestrutura e rodamos os servidores de desenvolvimento local. Agora estamos totalmente equipados para começar a construir e contribuir com o projeto.
